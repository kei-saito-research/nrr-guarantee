The current `priority-resolution` package supports a bounded tested-provider-pair account with four pieces: ambiguity-turn gain, downstream preacceptance-burden gain, repaired first-step side-topic / quarantine gain, and a conditional later-horizon boundary. The side-topic gain is not uniform across templates: baseline is already fairly strong on `A_trip_planning` and `B_hiring_packet`, while the main incremental lift over baseline is concentrated in `C_incident_response`. Later-horizon behavior remains conditional rather than uniformly stable, and the package should not be widened into provider-universal or Guarantee-style language.
